# Claude Code 读 PDF 公式乱码？这个开源 skill 一招解决

用 Claude Code 读论文，公式全变成 `/C22 /C21 /C28`？这不是你的问题，是 pypdf 的锅。

Claude Code 默认的 pdf skill 底层用的是 pypdf + pdfplumber，纯文本文档够用，但一遇到数学公式、双栏排版、扫描件就跪了——公式乱码、图片提不出、阅读顺序全乱。

**MinerU PDF** 就是解决这个的。它是一个 Claude Code 独立 skill，底层接入上海 AI Lab 开源的 MinerU 文档解析引擎，专门处理复杂 PDF。

说人话：**你告诉 Claude「解析这篇论文」，它自动切到 MinerU 引擎，公式变 LaTeX、图片带坐标、表格结构化输出。**

---

## 实测对比（同一篇双栏学术论文，7 页）

| 场景 | pypdf | MinerU |
|---|---|---|
| 公式 | 乱码 `/C22/C21...` | 可编译 LaTeX |
| 图片 | 无 | 21 张图片 + bbox 坐标 + 图注 |
| 输出 | 纯文本 | Markdown + JSON |

---

## 三步搞定

```bash
# 1. 安装 MinerU 引擎
pip install "mineru[pipeline]"

# 2. 安装 skill（Windows PowerShell）
New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.claude\skills\mineru-pdf"
Invoke-WebRequest -Uri "https://raw.githubusercontent.com/hawkongz/mineru-pdf/master/SKILL.md" -OutFile "$env:USERPROFILE\.claude\skills\mineru-pdf\SKILL.md"

# 3. 重启 Claude Code，输入
/mineru-pdf 解析这个 PDF
```

首次运行会自动下载模型（约 2GB），之后完全本地运行，不需要任何 API Key，不花钱。
